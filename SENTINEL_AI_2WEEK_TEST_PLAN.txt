════════════════════════════════════════════════════════════
  KNOWN ISSUES / NOTES
════════════════════════════════════════════════════════════

  1. Disk fill sim won't produce large disk_percent spikes on
     machines with >500 GB free — this is expected (200 MB write).

  2. Security threats are synthetic (4% chance per 30s scan).
     If no toast appears after 2 minutes, trigger manually via:
       curl -X POST http://localhost:5001/api/simulate/start/power_sag

  3. First anomaly detection takes ~3 minutes after start
     (warmup gate + baseline settling).

  4. Ollama must be running for AI diagnosis:
       brew services start ollama
     Without Ollama, diagnosis falls back to rule-based (still works).

  5. LSTM autoencoder trains after ~6.5 minutes (60+ sequences).
     Multivariate ml_isolation_forest anomalies fire sooner (~2 min).

  6. Multiple parallel main.py instances → incident flood.
     Always pkill -f "python.*main.py" before restarting.

  7. Remote stress_disk test may take up to 3s to clean up temp file
     on slow disks — this is expected behaviour.
